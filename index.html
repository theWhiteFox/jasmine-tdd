What is TDD?

According to Kent Beck...

  1. Don't write a line of new code unless you first have a failing automated test.
  2. Eliminate duplication.

Why practice TDD?

Documents behavior of code

  • Easy for new developers to join project
  • Stays up to date, unlike comments

Why practice TDD?

Leads to simple design

  • Solution uses minimal amount of code
  • Tests are first client of code

Why practice TDD?

Allows refactoring with confidence

  • Regressions are uncommon
  • Code improves over time

Why practice TDD?

Reduces number of defects

  • Maintainable code
  • Deploy any time

How to practice TDD?

Red Green Refactor

Write a short failing test

  • Watch your test fail

Make the test pass in the simplest way possible

  • Nothing too fancy
  • Duplicate code, if needed

Eliminate duplication introduced in the previous step

  • Improve code without changing behavior
  • Covered by tests
  • Don't forget!

Testing Asynchronous Code

Jasmine passes a done function as an argument to each function passed to it. 


it('takes done as an argument', function (done) {

});

Testing Asynchronous Code

A test will not pass until done has been called. 


it('takes done as an argument', function (done) {
  done();
});

Testing Asynchronous Code

By default, Jasmine will wait five seconds for done to be called before failing a test. 


it('will pass', function (done) {
  setTimeout(function () {
    done();
  }, 4000);
});

it('will fail', function (done) {
  setTimeout(function () {
    done();
  }, 6000);
});

Dependency injection

Angular uses dependency injection to provide objects with collaborators.


angular.module('messaging')
  .controller('messaging.flashController', function($scope,
                                                  messagesService) {

  });

Dependency injection

We can inject $scope in tests


describe('messages.flashController', function () {
  var $scope;
  beforeEach(module('messaging'));

  beforeEach(inject(function ($rootScope, $controller) {
    $scope = $rootScope.$new();



    $controller('messages.flashController', {
      $scope: $scope

    });
  }));

  // tests go here
});

Dependency injection

We can also inject our own mock objects.


describe('messages.flashController', function () {
  var $scope, mockMessagingService;
  beforeEach(module('messaging'));

  beforeEach(inject(function ($rootScope, $controller) {
    $scope = $rootScope.$new();
    mockMessagingService = jasmine.createSpyObj(['getMessage']);
    mockMessagingService.getMessage.and.return('Welcome!');

    $controller('messages.flashController', {
      $scope: $scope,
      messagingService: mockMessagingService
    });
  }));

  // tests go here
});
